# Make run detail pages readable and intentional across screen sizes

## Type

AFK

## User stories covered

10, 11, 12, 13, 14, 15

## What to build

Refine run detail pages so before-and-after inspection remains readable on narrow and wide screens while preserving the dashboard's calm visual tone. Keep stacked comparisons as the default presentation, allow a wider side-by-side comparison mode when space permits, and treat redacted, withheld, and unavailable content as deliberate interface states rather than placeholder-like gaps.

## Acceptance criteria

- [ ] Run detail pages use a stacked comparison layout by default and support a side-by-side presentation on wider screens.
- [ ] Long input and output content remains readable without cramped or noisy presentation.
- [ ] Redacted, withheld, empty, and unavailable detail states look intentional and understandable rather than broken.
- [ ] Any distinctive display typography remains optional and narrowly scoped so primary reading content stays neutral and legible.

## Blocked by

None - can start immediately.
