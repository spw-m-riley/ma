# Make recent-run rows compact, scannable, and status-aware

## Type

AFK

## User stories covered

2, 4, 5, 6, 12, 14

## What to build

Turn the recent-runs section into a compact list/table hybrid optimized for scanning and comparison. Align command, status, time, and short summary fields so a user can compare rows quickly, make failed runs stand out clearly, keep successful runs visually quiet, and preserve a calm overall surface even when the dashboard is busy.

## Acceptance criteria

- [ ] Recent-run rows present aligned command, status, time, and summary fields in a compact, easy-to-compare layout.
- [ ] Failed runs are visibly distinct without making the whole table feel loud.
- [ ] Successful runs remain visually quiet, and active runs are noticeable without dominating the page.
- [ ] The recent-runs layout stays readable and scannable across different data volumes and viewport widths.
