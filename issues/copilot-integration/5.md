# Dashboard integration for extension-mediated runs

Type: AFK

## What to build

Route stats from extension-mediated reduction runs into the existing dashboard observation system, so that invisible extension usage appears in the same history and summary views as normal CLI runs.

The extension tools already produce results in the standard `app.Result` contract. This slice makes the dashboard aware of extension-originated runs by recording them through the same observation path the CLI uses. The dashboard should distinguish extension runs from CLI runs (e.g. via a source field) while presenting them in unified summaries.

## Acceptance criteria

- [ ] Extension-mediated runs record stats to the dashboard history store
- [ ] Stats use the same contract as normal CLI runs
- [ ] Dashboard summary views include extension runs alongside CLI runs
- [ ] Extension runs are distinguishable from CLI runs in the history
- [ ] Stats are recorded without requiring additional user configuration

## Blocked by

- Issue 1 — Smart-read extension tool (extension must produce stats to record)
