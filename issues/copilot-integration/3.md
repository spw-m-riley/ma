# Rewrite skill for proactive agent behavior and terse responses

Type: AFK

## What to build

Replace the current manual-invocation SKILL.md with proactive agent rules that make the integration invisible. The rewritten skill should instruct the agent to:

- Prefer the smart-read tool for files it reads for understanding
- Use the standard file-reading tools for files it intends to edit, so that edit operations remain accurate
- Skip reduction for files shorter than approximately 200 lines
- Produce terser responses during sessions to reduce output token consumption, following the same general approach as output-reduction skills in the ecosystem

The terse-response instructions are prompt-engineering rather than deterministic transformation. Their effectiveness will vary by model and should be treated as best-effort.

The current manual-invocation guidance should be preserved as a secondary reference or folded into the extension's tool descriptions so it remains accessible for users who want explicit control.

## Acceptance criteria

- [ ] SKILL.md contains proactive rules rather than manual-invocation workflows
- [ ] Skill instructs the agent to use smart-read for understanding reads and standard tools for edit reads
- [ ] Skill includes a size threshold (~200 lines) below which reduction is skipped
- [ ] Skill includes terse-response instructions for reducing agent output verbosity
- [ ] Previous manual-invocation guidance is preserved or folded into tool descriptions
- [ ] The skill's threshold model produces the expected tool-choice behavior when tested against representative scenarios
