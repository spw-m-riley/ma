# Batch instruction maintenance command

Type: AFK

## What to build

Introduce a new CLI command (e.g. `ma maintain`) that walks an instruction-file tree, applies prose compression and duplicate detection across the files, and reports aggregate savings. This gives users a way to reduce the system-prompt payload before sessions start, in one pass.

The command should follow the same write contract as existing mutating commands: read-only by default, with `--write` to apply changes via `app.WriteWithBackup()` (creating `.ma.bak` backups). It should also support `--json` for structured output.

The tree-walking behavior should respect the same sensitive-path protections as individual file reads and handle mixed file types within an instruction directory gracefully (applying compression only to files classified as NaturalLanguage, skipping others).

## Acceptance criteria

- [ ] Command walks a directory tree and processes instruction files
- [ ] Prose compression and duplicate detection are applied across the file set
- [ ] Aggregate savings are reported accurately (total files, files changed, total token reduction)
- [ ] Read-only by default; `--write` applies changes with `.ma.bak` backups
- [ ] `--json` produces structured output in the standard `app.Result` contract
- [ ] Sensitive paths within the tree are rejected
- [ ] Non-NaturalLanguage files in the tree are skipped gracefully
- [ ] Command follows the one-package-per-command convention (`internal/maintain/`)

## Blocked by

None — can start immediately.
