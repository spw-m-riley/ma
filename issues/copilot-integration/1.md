# Smart-read extension tool with classification, reduction, and fallback

Type: AFK

## What to build

Build a Copilot CLI extension that ships in the `ma` repository and exposes a smart-read tool the agent can call during sessions. The tool accepts a file path, classifies it using the existing `detect` package, applies the matching reduction command (`compress` for prose, `skeleton` for code, `minify-schema` for config), and returns reduced content with stats in the standard `app.Result` contract.

If classification fails, the reduction command errors, or the `ma` binary is unavailable, the tool falls back to returning raw file content without surfacing an error to the agent. Files shorter than approximately 200 lines pass through without reduction.

The tool must respect the same sensitive-path protections as the CLI — calling `detect.IsSensitivePathResolved()` before reading any file and rejecting paths the CLI would reject.

This is the foundational slice for the Copilot CLI integration. It proves the full path: extension loads → agent calls tool → file classified → reduction applied → reduced content returned with stats → fallback works on failure.

The extension format and packaging conventions for the Copilot CLI should be verified against current documentation before implementation begins, since the extension system may have constraints on tool naming, discovery, or binary invocation that affect the design.

## Acceptance criteria

- [ ] Extension loads in the Copilot CLI and exposes a smart-read tool
- [ ] Smart-read classifies representative file types correctly (NaturalLanguage, Code, Config) and applies the expected reduction strategy for each
- [ ] Files shorter than ~200 lines pass through without reduction
- [ ] Smart-read falls back to raw content when classification fails, when reduction errors, and when the binary is unavailable, without surfacing errors to the caller
- [ ] Sensitive paths are rejected with the same behavior as the CLI (`detect.IsSensitivePathResolved()`)
- [ ] Tool output uses the standard `app.Result` contract with stats
- [ ] Fallback behavior is covered by tests, not just assumed
