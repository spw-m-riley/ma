# Realistic test fixtures and reduction threshold gates

## Type

AFK

## User stories covered

9, 11, 17, 18

## What to build

Add golden test fixtures that represent real-world compression workloads and assert meaningful minimum reduction thresholds. This is the final end-to-end verification that the full rule set meets its effectiveness targets.

Add at least four new fixture pairs:
- A verbose LLM-generated system prompt (heavy hedging, subject preambles, wordy phrases)
- A technical API documentation section (long-form technical terms, relative clauses)
- A project README (mixed prose with setup instructions, configuration references)
- An LLM-generated instructional guide (verbose, conversational, filler-heavy)

Each fixture should have a corresponding expected output and a minimum reduction threshold assertion. Verbose fixtures should assert ≥20% token reduction. Terse fixtures (if any are added) should assert ≥3%.

Additionally, run end-to-end structural preservation checks on all new fixtures using the existing validator to confirm that headings, code blocks, URLs, paths, and bullet counts survive compression intact.

## Acceptance criteria

- [ ] At least four new realistic golden-fixture pairs are added representing distinct real-world content types.
- [ ] Verbose fixtures assert ≥20% token reduction.
- [ ] Any terse fixtures assert ≥3% token reduction.
- [ ] All new fixtures pass structural preservation checks via the existing validator: headings, code blocks, URLs, paths, and bullet counts are preserved.
- [ ] Compressed output remains grammatically coherent and human-readable (verified by manual inspection of expected outputs).
- [ ] The test suite includes an aggregate threshold assertion across all realistic fixtures of ≥15% token reduction.
- [ ] All tests pass, confirming the full rule set delivers on the PRD's effectiveness targets.
