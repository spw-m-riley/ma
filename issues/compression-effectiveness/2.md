# Subject elision and hedging removal rules

## Type

AFK

## User stories covered

3, 14

## What to build

Add Phase 2 (subject elision, ~6 rules) and Phase 3 (hedging/softener removal, ~8 rules) to the phased compression engine.

Subject elision removes redundant second-person subjects from imperative-style instructions: `"You should always"` → (empty), `"You will need to"` → (empty), `"You must"` → `"must"`. These rules depend on contractions having already run in Phase 1, so `"You shouldn't"` is safe from elision while `"You should"` is correctly removed.

Hedging removal strips softening language that adds tokens without adding meaning: `"Please note that"` → (empty), `"It is important to note that"` → (empty), `"Be sure to"` → (empty), `"Make sure to"` → `"ensure"`. These patterns are especially common in LLM-generated instruction text.

Each category should have per-category unit tests confirming correct transformations and, for subject elision specifically, confirming that contracted negation forms are not matched.

## Acceptance criteria

- [ ] Subject elision rules (~6) are implemented as Phase 2 and correctly remove redundant subjects from imperative instructions.
- [ ] Hedging/softener removal rules (~8) are implemented as Phase 3 and correctly strip softening preambles.
- [ ] Subject elision does not match contracted negation: `"You shouldn't delete files"` remains unchanged after Phase 2.
- [ ] Per-category unit tests exercise representative patterns for both categories.
- [ ] Running `ma compress` on text containing hedging phrases like `"Please remember to"` and `"It is recommended to"` produces measurably shorter output.
- [ ] Zone protection and structural preservation remain intact.
